# FreeRadio — додаток для NVDA

FreeRadio — це додаток для прослуховування інтернет-радіо, призначений для екранного зчитувача NVDA. Його головна мета — надати користувачам простий доступ до тисяч інтернет-радіостанцій. Весь інтерфейс та всі функції розроблені з урахуванням повної доступності для NVDA.

## Каталог Браузер радіостанцій

FreeRadio використовує відкриту базу даних [Браузер радіостанцій](https://www.radio-browser.info/) для свого каталогу станцій. Браузер радіостанцій — це безкоштовний каталог, яким керує спільнота і який містить понад 50 000 інтернет-радіостанцій з усього світу. Реєстрація чи обліковий запис не потрібні, а його API відкритий для всіх. Кожна станція містить інформацію про адресу, країну, жанр, мову та бітрейт; станції сортуються за вподобаннями користувачів. FreeRadio підключається до цього API через дзеркальні сервери, розташовані в Німеччині, Нідерландах та Австрії; якщо один сервер недоступний, він автоматично перемикається на наступний.

## Додавання станції до Браузера радіостанцій

Якщо станція, яку ви шукаєте, відсутня в каталозі Браузера радіостанцій, ви можете додати її самостійно за адресою [https://www.radio-browser.info/add](https://www.radio-browser.info/add). Обліковий запис чи реєстрація не потрібні.

Заповніть форму на цій сторінці:

- **URL потоку** *(обов'язково)* — прямий URL аудіопотоку, що закінчується на `.mp3`, `.aac`, `.ogg` або подібне. Це не адреса веб-сайту станції; це адреса самого потоку, яку ви вставляєте в медіаплеєр. Більшість станцій публікують URL свого потоку на своєму веб-сайті або в розділі «Слухати в прямому ефірі».
- **Назва станції** *(обов'язково)* — назва станції, як вона має з'являтися в каталозі.
- **Головна сторінка** — адреса веб-сайту станції.
- **Країна та мова** — виберіть країну та мову з комбінованих списків.
- **Теги** — ключові слова жанру або теми, розділені комами, наприклад `новини`, `джаз`, `класика`. Вони використовуються для пошуку та фільтрування.
- **URL логотипу** — пряме посилання на зображення логотипу станції, якщо воно є.

Після надсилання станція проходить перевірку та додається до публічного каталогу. Після затвердження вона автоматично з’явиться у пошуку FreeRadio та списках країн, оскільки каталог оновлюється з API в режимі реального часу.

## Вимоги

- NVDA 2024.1 або пізніша версія
- Windows 10 або пізніша версія
- Підключення до Інтернету

## Встановлення

Завантажте файл `.nvda-addon`, натисніть на ньому Enter і перезапустіть NVDA, коли буде запропоновано.

## Комбінації клавіш

Усі комбінації можна перепризначити в меню NVDA → Параметри → Жести вводу → FreeRadio. Ці комбінації працюють будь-де, незалежно від того, який діалог сфокусований.

| Комбінація | Функція | Опис |
|---|---|---|
| `Ctrl+Win+R` | Відкрити браузер радіостанцій | Відкриває діалог браузера радіостанцій, якщо він закритий, або виводить його на передній план, якщо він вже відкритий. |
| `Ctrl+Win+P` | Пауза / відновлення | Зупиняє поточну станцію, якщо вона відтворюється; відновлює, якщо була призупинена. Якщо нічого не відтворюється, запускає останню станцію або відкриває список улюблених залежно від ваших налаштувань. Подвійне натискання одразу переходить до вибраної вкладки. Триразове натискання виконує окрему дію залежно від вашого налаштування. |
| `Ctrl+Win+S` | Зупинити | Повністю зупиняє поточну станцію та скидає плеєр. |
| `Ctrl+Win+→` | Наступна улюблена | Переходить до наступної станції у списку улюблених. У кінці списку переходить на початок. |
| `Ctrl+Win+←` | Попередня улюблена | Переходить до попередньої станції у списку улюблених. На початку списку переходить у кінець. |
| `Ctrl+Win+↑` | Збільшити гучність | Збільшує гучність на 5; максимум 200. |
| `Ctrl+Win+↓` | Зменшити гучність | Зменшує гучність на 5; мінімум 0. |
| `Ctrl+Win+V` | Додати до улюблених | Додає станцію, що зараз відтворюється, до списку улюблених. Повідомляє, якщо станція вже є у списку. |
| `Ctrl+Win+I` | Інформація про станцію | Повідомляє назву станції, що зараз відтворюється. Натисніть двічі, щоб у діалозі показати деталі, такі як країна, жанр та бітрейт. Натисніть чотири рази, щоб примусово запустити розпізнавання музики у разі неправильних метаданих ICY. |
| `Ctrl+Win+M` | Дзеркальний показ аудіо | Одночасно дзеркально показує потік на додатковий пристрій виведення аудіо. Натисніть ще раз, щоб зупинити дзеркальний показ. |
| `Ctrl+Win+E` | Миттєвий запис | Натисніть один раз, щоб почати запис поточної станції; натисніть ще раз, щоб зупинити. Натисніть **двічі**, щоб почати **запис треку** — файл отримає назву поточного треку, а запис автоматично зупиниться, коли трек зміниться. Натисніть двічі, поки запис треку активний, щоб зупинити його достроково. Відтворення продовжується без перерв у всіх режимах запису. Доступно лише для станцій, що транслюють метадані ICY. |
| `Ctrl+Win+W` | Відкрити папку записів | Відкриває папку із записаними файлами у Провіднику файлів. |
| `Ctrl+Win+J` | Тайм-шифт — перемотування назад | Перемотує живе радіо на 15 секунд назад. Перше натискання активує режим тайм-шифту; кожне наступне перемотує ще на 15 секунд назад, до межі буфера (~10 хвилин). Потребує увімкненого буфера тайм-шифту в Налаштуваннях. |
| `Ctrl+Win+K` | Тайм-шифт — перемотування вперед | Перемотує на 15 секунд вперед у режимі тайм-шифту. Досягнувши краю прямого ефіру, відтворення автоматично повертається до прямого ефіру, і команда не має ефекту до наступного перемотування назад. |
| `Ctrl+Win+T` | Увімкнути/вимкнути буфер тайм-шифту | Миттєво вмикає або вимикає буфер тайм-шифту, відображаючи стан прапорця в Налаштуваннях. При вимкненні негайно повертає до прямого ефіру (якщо активний тайм-шифт) та зупиняє фонове захоплення. |
| *(непризначено)* | Увімкнути/вимкнути сповіщення про вимкнення звуку | Увімкнути або вимкнути налаштування «Сповіщення про вимкнення звуку» миттєво. Призначте комбінацію клавіш через меню NVDA → Параметри → Жести вводу → FreeRadio. |

Клавіші «Наступний» та «Попередній» призначені лише для навігації по списку улюблених станцій; вони не працюють зі списком усіх станцій. Коли у діалозі браузера радіостанцій виділено список, клавіші зі стрілками вліво та вправо виконують одну й ту саму функцію — див. «Клавіші швидкого доступу в діалогах».

## Браузер радіостанцій

FreeRadio також додає підменю **FreeRadio** до меню «Інструменти» NVDA. Звідти ви можете безпосередньо відкрити браузер станцій та налаштування FreeRadio.

Діалог, що відкривається за допомогою комбінації клавіш `Ctrl+Win+R`, містить п’ять вкладок: «Усі станції», «Улюблене», «Запис», «Таймер» та «Улюблені треки». Переміщатися між вкладками можна за допомогою комбінації клавіш `Ctrl+Tab`.

Коли відкривається вкладка «Усі станції», 1000 станцій, що набрали найбільшу кількість вподобань, автоматично завантажуються з браузера радіостанцій. Вибір країни зі списку оновлює список, показуючи станції цієї країни. Введення тексту в поле пошуку миттєво  виконує повний пошук по всій базі даних браузера радіостанцій одночасно за назвою, країною та жанром.

Комбінований список **Пристрій виводу** у нижній частині діалога браузера радіостанцій — поза вкладками — містить список усіх аудіопристроїв, розпізнаних BASS. Вибір пристрою негайно перенаправляє аудіовивід на нього та зберігає вибір назавжди; той самий пристрій автоматично використовується в наступній сесії. Якщо вибраний пристрій не підключено, додаток автоматично повертається до системного значення. Цей елемент керування працює лише тоді, коли активний бекенд BASS.

Елементи керування **Гучність** (0–200) та **Ефекти** у цій же області можна налаштовувати в будь-який час, поки діалог відкритий. У списку ефектів можна одночасно ввімкнути хор, компресор, спотворення, ехо, фленджер, булькання, реверберацію, еквалайзер: підсилення низьких частот, еквалайзер: підсилення високих частот та еквалайзер: підсилення вокалу; зміни миттєво застосовуються до активного потоку. Ці елементи керування повністю функціонують лише тоді, коли активний бекенд BASS.

Коли ввімкнено один або кілька ефектів еквалайзера, для кожної активної смуги з’являється **регулятор підсилення**. Рівень посилення можна встановити в діапазоні від −15 дБ до +15 дБ; початкове значення — «Баси» +9 дБ, «Високі частоти» +9 дБ та «Вокал» +6 дБ. Регулятори посилення відображаються лише для тих смуг еквалайзера, які наразі позначені, і автоматично приховуються, коли ефекти еквалайзера знімаються. Значення посилення зберігаються глобально та відновлюються під час наступної сесії.

Кнопка **Відтворення/Пауза** також розташована внизу діалога. Якщо жодна станція не відтворюється, вона запускає вибрану станцію; якщо станція вже відтворюється, вона призупиняє відтворення.

Коли станція вибрана у списку, кнопка **Інформація про станцію** показує таку інформацію, як країна, мова, жанр, формат, бітрейт, веб-сайт та URL потоку в окремому діалозі. Кожне поле показується у власному текстовому полі, доступному лише для читання; ви можете переходити між полями за допомогою клавіші Tab та скопіювати всю інформацію до буфера обміну одночасно за допомогою кнопки **Копіювати все до буфера обміну**. Ця кнопка доступна як на вкладці «Усі станції», так і на вкладці «Улюблене».

### Комбінації клавіш у діалозі

Наступні клавіші працюють лише тоді, коли діалог Браузера радіостанцій активний.

### Клавіші F

| Комбінація | Функція | Опис |
|---|---|---|
| `F1` | Довідка | Відкриває файл довідки додатка у браузері. Спочатку здійснюється пошук довідки для мови, яка використовується в NVDA; якщо її не знайдено, відкривається довідка початковою мовою. |
| `F2` | Що зараз відтворюється | Оголошує назву станції та треку, що зараз відтворюються. Натисніть двічі, щоб у діалозі показати детальну інформацію, таку як країна, жанр та бітрейт. Натисніть тричі, щоб скопіювати інформацію про поточний трек (метадані ICY) до буфера обміну, якщо вона доступна; якщо метадані відсутні, замість цього запускається розпізнавання треків Shazam. Натисніть чотири рази, щоб примусово запустити розпізнавання музики у разі неправильних метаданих ICY. |
| `F3` | Попередня станція | Переходить до попередньої станції на вкладці «Усі станції» або «Улюблене» та негайно починає відтворення. Переходить до кінця, якщо знаходиться на початку списку. |
| `F4` | Наступна станція | Переходить до наступної станції у вкладці «Усі станції» або «Улюблене» та негайно починає відтворення. Переходить до початку, якщо знаходиться в кінці списку. |
| `F5` | Зменшити гучність | Зменшує гучність на 5 (мінімум 0). |
| `F6` | Збільшити гучність | Збільшує гучність на 5 (максимум 200). |
| `F7` | Пауза / продовження | Призупиняє відтворення, якщо станція відтворюється; продовжує відтворення, якщо вона була призупинена і медіа завантажено. |
| `F8` | Зупинка | Повністю зупиняє поточну станцію та скидає плеєр. |
| `F9` | Перейменувати | Відкриває діалог перейменування для станції, що знаходиться у фокусі на вкладці «Улюблене». |

## Улюблене

Список улюблених — це особиста колекція радіостанцій, яка зберігається постійно. Щоб додати радіостанцію, виберіть її у списку та натисніть кнопку «Додати до улюблених» або скористайтеся комбінацією клавіш `Alt+V`. Ця ж комбінація клавіш видаляє радіостанцію, яка вже є у списку, після її вибору.

До улюбленого можна перейти за допомогою комбінацій клавіш `Ctrl+Win+→` та `Ctrl+Win+←`; ці комбінації працюють навіть тоді, коли діалог браузера радіостанцій закритий.

Щоб видалити станцію зі списку улюблених, виберіть її та натисніть кнопку **Видалити станцію** або клавішу `Delete`. Після видалення фокус та вибір автоматично переходять до наступної станції у списку. Якщо видалена станція була останньою, фокус переходить до попередньої станції. Якщо список стає порожнім, фокус переходить до кнопки «Відтворити».

### Експорт та імпорт улюблених

Вкладка «Улюблене» містить дві кнопки для резервного копіювання та відновлення списку станцій:

**Експортувати улюблені…** — зберігає весь список улюблених у файл. У діалозі збереження можна вибрати один із двох форматів:
- **JSON** (`.json`) — повна резервна копія, що зберігає назви станцій, URL-адреси потоків і всі метадані. Рекомендується для подальшого відновлення списку або його перенесення на інший комп'ютер.
- **Плейлист M3U** (`.m3u`) — стандартний формат плейлиста, сумісний із більшістю медіаплеєрів і радіододатків. Зверніть увагу, що M3U не зберігає всі метадані станцій, тому відновлення з M3U може містити менше деталей, ніж резервна копія JSON.

**Імпортувати улюблені…** — завантажує станції з раніше експортованого файлу JSON або M3U. Після вибору файлу пропонується вибрати спосіб додавання станцій:
- **Так (Об'єднати)** — додає імпортовані станції до наявного списку, не видаляючи поточні улюблені. Дублікати не додаються повторно.
- **Ні (Замінити)** — повністю очищає поточний список улюблених і замінює його вмістом імпортованого файлу.
- **Скасувати** — повертається до браузера без внесення жодних змін.

Після успішного імпорту список улюблених, список станцій для запланованих записів і список станцій таймера оновлюються автоматично.

### Зміна порядку в списку улюблених

Вибравши станцію на вкладці «Улюблене», натисніть клавішу «кома», щоб увійти в режим переміщення — пролунає звуковий сигнал. Перейдіть до потрібної позиції за допомогою клавіш зі стрілками, а потім знову натисніть клавішу «кома». Станція буде розміщена у вибраній позиції, а нове розташування одразу збережеться. Повторне натискання клавіші «кома» у цій же позиції скасує переміщення.

### Додавання користувацької станції

Щоб додати станцію, якої немає в Браузері радіостанцій, скористайтеся кнопкою «Додати користувацьку станцію». У діалозі, що з’явиться, введіть назву станції та URL потоку, щоб додати її безпосередньо до списку улюблених. Власні станції можна відтворювати та змінювати їхній порядок так само, як і будь-які інші улюблені станції.

### Аудіопрофіль станції

На вкладці «Улюблені» є дві кнопки для керування налаштуваннями звуку для кожної станції:

**Зберегти аудіопрофіль для цієї станції** — зберігає поточний рівень гучності, активні ефекти (хор, еквалайзер тощо) та значення підсилення еквалайзера як профіль, прив’язаний до цієї конкретної станції. Щоразу, коли починається відтворення цієї станції, її збережені налаштування гучності, ефектів та підсилення автоматично застосовуються, замінюючи загальні початкові налаштування .

**Очистити аудіопрофіль** — видаляє збережений аудіопрофіль з вибраної станції. Після очищення станція повертається до загальних налаштувань гучності, ефектів та підсилення еквалайзера. Ця кнопка активна лише тоді, коли вибрана станція вже має збережений профіль.

Обидві кнопки розташовані під списком улюблених і стають активними лише тоді, коли у списку вибрано станцію.

## Розпізнавання треків

Тричі натиснувши `Ctrl+Win+I`, запускається розпізнавання треків на основі Shazam для потоку, що відтворюється. Розпізнавання починається лише тоді, коли метадані ICY (інформація про трек, що транслюється станцією) недоступні; якщо метадані присутні, вони копіюються в буфер обміну.

Розпізнавання працює наступним чином: за допомогою ffmpeg із потоку витягується короткий аудіофрагмент, до нього застосовується алгоритм Shazam для визначення «відбитка», а результат надсилається на сервери Shazam. Якщо розпізнавання проходить успішно, назва треку, виконавець, альбом та рік випуску оголошуються NVDA та автоматично копіюються до буфера обміну. Якщо увімкнено опцію **Зберегти вподобані пісні у текстовий файл**, результат розпізнавання також додається до файлу `likedSongs.txt`.

**Аудіосигнали:** Два висхідних звукових сигнали лунають, коли розпізнавання починається, і два низхідних — коли воно закінчується. Короткий звуковий сигнал відтворюється кожні 2 секунди, доки процес триває.

**Вимога:** Потрібна наявність ffmpeg.exe. Якщо ffmpeg.exe знаходиться у папці додатка, він використовуватиметься автоматично; якщо в іншому місці — шлях можна вказати в Налаштуваннях. Завантажити ffmpeg можна з [ffmpeg.org](https://ffmpeg.org/download.html).

## Дзеркало аудіо

Комбінація клавіш `Ctrl+Win+M` дозволяє одночасно відтворювати потокове аудіо на другому пристрої відтворення. Це зручно, коли потрібно слухати аудіо одночасно на двох різних пристроях, наприклад на колонках і навушниках.

Після першого натискання з’являється діалог з переліком доступних пристроїв виведення. Як тільки пристрій обрано, починається дзеркальне відтворення, при цьому основне відтворення не переривається. Повторне натискання зупиняє дзеркалювання.

**Приклади використання:**
- **Динаміки + навушники** — Гість може слухати той самий потік в навушниках, поки ви слухаєте його через динаміки комп’ютера.
- **Налаштування запису** — Направте сигнал головного виходу на колонки, а сигнал другого виходу — на зовнішній записний пристрій або аудіоінтерфейс для зовнішнього запису.
- **Multi-room** — Відтворюйте треки одночасно через Bluetooth-колонку та вбудований динамік; для передачі звуку в іншу кімнату не потрібно додаткового програмного забезпечення.
- **Дистанційний моніторинг** — під час сеансу спільного використання екрана або віддаленого доступу до робочого столу як локальний, так і віддалений користувач можуть одночасно слухати один і той самий аудіопотік.

> **Примітка:** Функція дзеркального аудіо доступна лише тоді, коли активний бекенд BASS. Якщо під час дзеркального відтворення змінити гучність, обидва виходи оновлюються одночасно.

## Запис

Записи початково зберігаються у папці `Documents\FreeRadio Recordings\`. Назва файлу містить назву станції (або назву трека у режимі запису трека) та час початку запису. Папку для записів можна змінити у будь-який час: Меню NVDA → Налаштування → FreeRadio → **Папка для записів**. Оскільки механізм запису підключається безпосередньо до потоку, аудіо записується на диск у тому вигляді, у якому його отримано — обробка або повторне кодування не застосовується; якість запису ідентична якості трансляції.

**Миттєвий запис:** під час відтворення станції натисніть Ctrl+Win+E один раз. Натисніть знову, щоб зупинити. Відтворення при цьому не переривається.

**Запис трека:** Натисніть комбінацію клавіш `Ctrl+Win+E` **двічі** під час відтворення станції, яка транслює метадані ICY. Запис розпочнеться негайно, а його назва буде відповідати назві поточного треку. Коли трек зміниться, запис автоматично зупиниться, і NVDA оголосить назву збереженого файлу. Якщо ви хочете завершити запис до закінчення треку, знову натисніть `Ctrl+Win+E` двічі. Якщо поточна станція не транслює метадані ICY, запис трека недоступний, і NVDA повідомить вас про це.

**Запланований запис:** Відкрийте вкладку Запис у браузері радіостанцій. Виберіть станцію з улюблених, введіть час початку у форматі ГГ:ХХ і тривалість у хвилинах, виберіть один або кілька активних днів, потім оберіть режим повторення та режим запису:

**Активні дні:** Позначте один або кілька днів тижня. У режимі одноразового запису для кожного вибраного дня створюється окремий запис, встановлений на найближче настання цього дня. У режимі повторення запис виконується лише у позначені дні. Якщо жодного дня не вибрано, запис не обмежується конкретними днями.

**Режим повторення:**
- **Записати один раз** — створює одноразовий запис для кожного вибраного дня. Кожен запис встановлюється на найближче настання цього дня; якщо сьогоднішній час вже минув, запис автоматично переноситься на наступний тиждень.
- **Повторювати щотижня** — повторюється щотижня у вибрані активні дні до видалення зі списку розкладу.

**Режим запису:**
- **Запис під час прослуховування** — одночасне відтворення та запис. Для відтворення використовується один із програвачів у порядку пріоритетності: BASS → VLC → PotPlayer → Windows Media Player.
- **Тільки запис** — запис у фоновому режимі без відтворення звуку; механізм запису підключається безпосередньо до потоку.

NVDA сповіщає про початок і завершення запису. Якщо NVDA перезапускається під час активного запланованого запису, запис автоматично відновлюється під час запуску.

## Тайм-шифт (перемотування живого радіо)

Тайм-шифт дозволяє перемотувати станцію, яку ви слухаєте, як DVR або касета — зупиніть момент, поверніться на кілька хвилин назад і доженіть прямий ефір, коли захочете. Відтворення при цьому не зупиняється: перемотування назад і вперед відбувається миттєво на тому ж аудіопотоці.

Функція **вимкнена за замовчуванням**. Увімкніть її в меню NVDA → Налаштування → Параметри → FreeRadio → **Увімкнути буфер тайм-шифту (перемотування прямого радіо, ~10 хвилин)**, або миттєво перемикайте будь-коли за допомогою `Ctrl+Win+T`.

### Як це працює

Після увімкнення FreeRadio безперервно захоплює відтворювану станцію у плаваючий локальний буфер у фоновому режимі. Буфер містить приблизно **останні 10 хвилин** звуку; старий звук автоматично видаляється з початку в міру надходження нового, тому буфер завжди представляє «недавнє минуле» відносно краю прямого ефіру.

- **`Ctrl+Win+J`** — Перемотати на 15 секунд назад. Перше натискання переключає вас з живого відтворення у режим тайм-шифту, починаючи з 15 секунд позаду краю прямого ефіру. Кожне наступне натискання перемотує ще на 15 секунд назад.
- **`Ctrl+Win+K`** — Перемотати на 15 секунд вперед у режимі тайм-шифту. Досягнувши краю прямого ефіру, відтворення автоматично переключається назад до прямого потоку, і NVDA оголошує «Повернутися до прямого ефіру».
- **`Ctrl+Win+T`** — Вмикає або вимикає всю функцію. Вимкнення в режимі тайм-шифту негайно повертає до прямого ефіру та зупиняє фонове захоплення для поточної станції.

Фонове захоплення продовжується весь час, поки активний тайм-шифт, тому край прямого ефіру продовжує рухатися вперед, навіть коли ви слухаєте щось кількахвилинної давнини — точно як справжній DVR.

### Увімкнення та прогрів буфера

Буфер починає заповнюватися, щойно станція починає грати (при увімкненій функції) або в момент, коли ви вмикаєте функцію під час прослуховування станції. Тому перемотування можливе лише після того, як реально накопичено кілька секунд звуку — якщо натиснути `Ctrl+Win+J` одразу після переключення станцій, NVDA повідомить, що в буфері поки недостатньо звуку. Просто зачекайте кілька секунд і спробуйте знову.

Переключення на іншу станцію завжди перезапускає буфер для нової станції; буферизований звук попередньої станції видаляється.

### Підтримувані потоки

Тайм-шифт працює з тим самим діапазоном потоків, які вже підтримує FreeRadio:

- Звичайні HTTP/HTTPS-потоки (MP3, AAC, OGG тощо), включаючи сервери у стилі Shoutcast/Icecast.
- **HLS-потоки (`.m3u8`)** — FreeRadio вирішує основний плейлист станції, відстежує медіаплейлист і завантажує сегменти у фоні, щоб буфер залишався заповненим.

У рідкісному випадку, коли плейлист станції взагалі неможливо прочитати (наприклад, пошкоджений або недоступний маніфест `.m3u8`), NVDA повідомить, що перемотування недоступне для цієї конкретної станції.

### Вимоги та обмеження

- **Потребує бекенду BASS.** Тайм-шифт недоступний, коли BASS вимкнено.
- Буфер розрахований приблизно на 10 хвилин; перемотати далі неможливо.
- Буфер прив'язаний до станції: зміна станції, зупинка відтворення або перезапуск NVDA очищають його й починають заново.
- Відтворення з тайм-шифтом використовує власний локальний файл буфера і не створює збереженого запису — якщо хочете зберегти звук назавжди, використовуйте також Миттєвий запис (`Ctrl+Win+E`).

## Таймер

Відкрийте вкладку «Таймер» у браузері радіостанцій (Alt+4). Можна додати два типи таймерів:

**Сигнал — увімкнути радіо:** Автоматично починає відтворення обраної станції з ваших улюблених у вказаний час. Оберіть станцію та введіть час у форматі ГГ:ХХ.

**Сон — зупинка радіо:** Зупиняє відтворення у вказаний час. Коли спрацьовує таймер, гучність поступово зменшується протягом 60 секунд, перш ніж відтворення зупиниться. Вибір станції не потрібен; просто введіть час.

Для обох типів, якщо вказаний час уже минув, дія планується на наступний день. Якщо на той самий час вже існує таймер (незалежно від типу), додавання нового заблоковано; користувач отримує повідомлення про конфлікт і пропозицію спочатку видалити наявний запис. Таймери, що очікують на виконання, перелічені у вкладці; виберіть один і натисніть кнопку «Видалити вибраний таймер», щоб скасувати його.

## Налаштування

Наступні параметри можна налаштувати в меню NVDA → Параметри → Налаштування → FreeRadio:

| Параметр | Опис |
|---|---|
| Пристрій виведення звуку (модуль BASS) | Встановлює пристрій виведення звуку для відтворення радіо. Список містить усі пристрої, сумісні з BASS, що є в системі, а також параметр «Стандартно, як у системі». Зміни застосовуються одразу після збереження; якщо вибраний пристрій відключено, додаток автоматично переходить на системний і повідомляє про це. Функціонує лише під час використання модуля BASS. |
| Гучність | Встановлює початкову гучність додатка (0–200). Зміни, внесені під час відтворення за допомогою клавіш `Ctrl+Win+↑` / `Ctrl+Win+↓`, також показуються тут. |
| Початковий Аудіоефект | Встановлює аудіоефект, що застосовується під час запуску NVDA або початку відтворення станції. Вибраний ефект відповідає списку «Ефекти» у браузері радіостанцій. Працює лише у разі використання бекенду BASS. |
| Посилення еквалайзера (низькі частоти / високі частоти / вокал) | Встановлює рівень посилення в дБ для кожної смуги еквалайзера (від -15 до +15). Ці значення застосовуються, коли активний відповідний ефект еквалайзера, і зберігаються глобально. Перевизначення для кожної станції можна зберегти за допомогою кнопки **Зберегти аудіопрофіль** на вкладці «Улюблене». Функція активна лише під час використання бекенду BASS. |

Не має ефекту та не впливає на продуктивність, якщо встановлено Миттєве перемикання. Доступно лише під час використання бекенду BASS. |
| Відновлювати останню станцію під час запуску NVDA | Якщо увімкнено, остання відтворювана станція автоматично відновлюється щоразу при запуску NVDA. |
| Автоматичне оголошення змін трека (метадані ICY) | Якщо цей параметр увімкнений, NVDA автоматично оголошує назву нового трека щоразу, коли він змінюється на станції, яка транслює метадані ICY. Назва першого трека також оголошується одразу після перемикання на нову станцію. Початково вимкнено. |
| Вимкнути сповіщення | Якщо увімкнено, NVDA не повідомляє про зміну станцій, зміну стану відтворення (відтворення, пауза, зупинка) або події запису (початок, зупинка, завершення). Повідомлення про помилки, відгуки про улюблене, результати розпізнавання треків та повідомлення про оновлення залишаються активними. Також можна перемикати миттєво за допомогою непризначеного жесту вводу. Початково вимкнено. |
| Увімкнути буфер тайм-шифту (перемотування прямого радіо, ~10 хвилин) | Вмикає або вимикає функцію тайм-шифту. При увімкненні відтворювана станція безперервно захоплюється у фоні, щоб її можна було перемотувати назад за допомогою `Ctrl+Win+J` та вперед за допомогою `Ctrl+Win+K`. Також можна миттєво перемикати за допомогою `Ctrl+Win+T`. Потребує бекенду BASS. Вимкнено за замовчуванням. |
| Збереження вподобаних треків у текстовий файл | Якщо цей параметр увімкнений, інформація про трек, скопійована в буфер обміну потрійним натисканням клавіш `Ctrl+Win+I`, також додається до файлу `Documents\FreeRadio Recordings\likedSongs.txt`. Якщо метадані ICY відсутні, до цього ж файлу зберігається результат розпізнавання Shazam. Початково цей параметр вимкнений. |
| Коли натиснуто Ctrl+Win+P без активного відтворення | Визначає, що відбувається при натисканні цієї комбінації клавіш, якщо нічого не відтворюється: запустити останню станцію або відкрити список улюблених. |

| Автоматична перевірка оновлень | Якщо цей параметр увімкнений, під час кожного запуску NVDA виконується фонова перевірка оновлень; у разі виявлення нової версії ви отримаєте відповідне повідомлення. Якщо цей параметр вимкнений, автоматична перевірка припиняється, але ручна перевірка залишається доступною. |
| Шлях до ffmpeg.exe | Шлях до файлу ffmpeg.exe, який використовується для розпізнавання треків. Якщо поле залишити порожнім, автоматично використовується файл ffmpeg.exe з папки додатка. |
| Шлях до VLC | Якщо VLC не встановлено або знаходиться в нестандартному місці, тут можна вказати повний шлях до виконуваного файлу. |

## Вимкнути сповіщення

Якщо в налаштуваннях увімкнено **Вимкнути сповіщення**, NVDA вимикає наведені нижче автоматичні повідомлення:

- Назва станції, коли починається відтворення нової станції
- Зміни стану відтворення: відтворення, пауза, зупинка
- Події запису: початок, зупинка, завершення (миттєвий запис, запис трека та запис за розкладом)
- Оголошення про зміну трека в ICY, навіть якщо також увімкнено функцію **Автоматичне оголошення про зміну трека**

Нижченаведені повідомлення навмисно **не** підпадають під ці зміни: повідомлення про помилки, відгуки щодо улюблених треків (додано / вже є у списку), результати розпізнавання треків та повідомлення про оновлення.

Цей параметр можна увімкнути або вимкнути в меню NVDA → Параметри → Налаштування → FreeRadio, або миттєво в будь-який момент за допомогою жесту вводу, якому не призначено жодної комбінації (призначити такий жест можна в меню NVDA → Параметри → Жести вводу → FreeRadio). Після увімкнення або вимкнення NVDA один раз озвучить повідомлення «Сповіщення вимкнено» або «Сповіщення увімкнено» для підтвердження зміни.

## Автоматичне повідомлення про відстеження змін

Якщо в налаштуваннях увімкнено параметр **«Автоматичне оголошення зміни треку»**, FreeRadio приблизно кожні 5 секунд у фоновому режимі перевіряє потік метаданих ICY поточної станції. Коли трек змінюється, NVDA автоматично зачитує назву нового трека — натискання клавіш не потрібне.

Під час перемикання на нову станцію інформація про перший трек оголошується одразу після встановлення з'єднання. Якщо ви перемикаєтеся на станцію, яка не транслює метадані ICY, система не видає жодних звукових сигналів, а інформація про трек попередньої станції не повторюється.

Ця функція початково вимкнена, і її можна перемикати в меню NVDA → Параметри → Налаштування → FreeRadio.

## Вподобані треки

Якщо увімкнений параметр **«Зберегти вподобані пісні у текстовий файл»**, інформація про треки, скопійована в буфер обміну потрійним натисканням клавіш `Ctrl+Win+I`, також додається по рядках до файлу `Documents\FreeRadio Recordings\likedSongs.txt`.

На радіостанціях, що транслюють метадані ICY, назва композиції та ім’я виконавця зберігаються безпосередньо. На радіостанціях без метаданих ICY результат розпізнавання Shazam зберігається в тому самому файлі — обидва джерела використовують один і той самий список. Якщо файл не існує, він створюється автоматично; кожен запис додається в кінець файлу, а попередні записи ніколи не видаляються.

## Вкладка «Улюблені треки»

Вкладка **«Улюблені треки»** в браузері станцій відображає всі треки, збережені у `likedSongs.txt`. Список автоматично перезавантажується з файлу кожного разу при відкритті вкладки.

Поле **Фільтр** над списком дозволяє звужувати відображувані треки в реальному часі. Введіть будь-яку частину назви пісні або імені виконавця — список миттєво оновлюється після кожного натискання клавіші. NVDA оголошує кількість відповідних результатів після кожної зміни. Натисніть стрілку `Вниз` у полі фільтра, щоб перемістити фокус безпосередньо до списку.

При виборі треку зі списку стають доступними такі дії:

- **Відтворити у Spotify:** Намагається безпосередньо відкрити настільний застосунок Spotify. Якщо застосунок не встановлено, переходить на сайт Spotify і автоматично починає відтворення першого результату.
- **Відтворити на YouTube (`Alt+O`):** Виконує пошук вибраного треку на YouTube і відкриває результати у браузері за замовчуванням.
- **Показати текст пісні:** Завантажує та відображає текст вибраного треку. Текст пісні завантажується з [lrclib.net](https://lrclib.net) (безкоштовно, без облікового запису). Під час пошуку у фоновому режимі оголошується коротке повідомлення «Завантаження тексту…». Якщо текст знайдено, він відкривається у діалозі лише для читання, де його можна читати за допомогою NVDA та копіювати до буфера обміну. Якщо текст не знайдено, NVDA повідомляє про це. Кнопка тимчасово вимикається під час завантаження, щоб запобігти дублюванню запитів.
- **Видалити (`Alt+M`):** Видаляє вибраний трек з `likedSongs.txt` і оновлює список. Клавіша `Delete` також активує цю кнопку, коли список знаходиться у фокусі.
- **Оновити (`Alt+E`):** Перезавантажує список із файлу.

Кнопки Spotify, YouTube, «Показати текст пісні» та «Видалити» активні лише тоді, коли у списку вибрано реальний трек.

### Служба текстів пісень

FreeRadio використовує [lrclib.net](https://lrclib.net) для отримання текстів пісень — безкоштовна відкрита база даних, яка не потребує ключа API або облікового запису. Процес пошуку аналізує рядок треку, збережений у `likedSongs.txt`, і послідовно пробує ширші запити до знаходження тексту:

1. Точний збіг із повним іменем виконавця та очищеною назвою (шумові суфікси, як-от «Remastered», «Live» або теги року, видаляються перед пошуком).
2. Точний збіг із повним іменем виконавця та оригінальною назвою (якщо очищення її змінило).
3. Точний збіг лише з іменем першого виконавця та очищеною назвою (для рядків із кількома виконавцями, наприклад «Виконавець A & Виконавець B»).
4. Нечіткий пошук із іменем першого виконавця та очищеною назвою.
5. Нечіткий пошук із вихідним рядком треку як крайній засіб.

Якщо доступні прості тексти, вони відображаються як є. Якщо доступні лише синхронізовані за часом LRC-тексти, мітки часу видаляються і відображається простий текст. Інструментальні треки повідомляються як не знайдені.


## Відтворення

Додаток вибирає механізм відтворення, дотримуючись такого порядку пріоритетів:

1. **BASS** — стандартний і основний бекенд. Окремого встановлення не потрібно; він входить до складу додатка. BASS надсилає аудіо безпосередньо до аудіостеку Windows і відображається в мікшері гучності Windows як самостійне джерело звуку під назвою «pythonw.exe», окреме від NVDA. Це означає, що аудіо FreeRadio передається по повністю окремому каналі від мовлення NVDA: радіо не переривається, не змішується та не зазнає впливу власних налаштувань аудіо NVDA під час мовлення NVDA. Користувач може регулювати гучність радіо незалежно від NVDA у мікшері гучності Windows. Підтримує HTTP, HTTPS та більшість вбудованих форматів потокового аудіо. Дзеркалення аудіо доступне лише з цим бекендом.
2. **VLC** — бере на себе функції, якщо BASS не працює. Автоматично шукається у типових місцях інсталяції, папках профілів користувачів та системному PATH.
3. **PotPlayer** — використовується, якщо VLC не знайдено. Автоматично шукається у типових місцях інсталяції.
4. **Windows Media Player** — використовується як крайній захід; вимагає інсталяції компонента WMP у системі.

## Перевірка оновлень

FreeRadio автоматично перевіряє наявність нових версій через GitHub.

**Автоматична перевірка:** працює тихо у фоновому режимі через 15 секунд після запуску NVDA. Якщо знайдено нову версію, ви отримаєте сповіщення; якщо нічого не знайдено, повідомлення не показується.

**Ручна перевірка:** Можна запустити за бажанням у меню NVDA «Інструменти» → «FreeRadio» → **«Перевірити наявність оновлень…»**. При запуску таким чином результат оголошується навіть у тому випадку, якщо версія є актуальною.

**Якщо оновлення знайдено:** Відкриється діалог з номером версії та встановленою версією.

- Якщо у випуску на GitHub доступний файл `.nvda-addon` для прямого завантаження, з’являється кнопка **Завантажити та встановити**. Після підтвердження файл завантажується у фоновому режимі, NVDA повідомляє про початок завантаження, а потім автоматично відкривається власний діалог встановлення NVDA.
- Якщо посилання для прямого завантаження відсутнє, з’являється кнопка **Відкрити сторінку**, і сторінка випуску на GitHub відкривається у браузері.

**Щоб вимкнути автоматичні перевірки:** Вимкніть параметр **Автоматично перевіряти наявність оновлень** у меню NVDA → Параметри → Налаштування → FreeRadio.
